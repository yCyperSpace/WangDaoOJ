# 需求文档

## 1. 项目目标

构建一个面向王道数据结构代码题的在线判题系统，支持用户在线编码、提交、查看结果，以及上传题目并通过 AI 生成参考程序。

## 2. 用户角色

- 普通用户：浏览题目、编写并提交代码
- 出题用户：上传题目、样例、生成参考解
- 管理员：维护题库、测试数据、系统参数

## 3. 核心功能

### 3.1 在线做题

- 浏览题目列表
- 查看题目描述、输入说明、输出说明、样例
- 在线编写 C++ 代码，并可选择 `C++11 / C++14 / C++17`
- `cin / cout / cerr / clog / endl` 需要有额外高亮
- 支持在正式提交前运行公开样例
- 运行结果需展示期望输出、实际输出、错误信息和差异
- 提交后获得 `accepted`、`wrong_answer`、`compile_error`、`runtime_error`、`time_limit_exceeded`

### 3.2 上传题目

- 用户只上传原始题面
- 系统调用 DeepSeek 生成题目标题、输入输出说明、参考解、公开样例和隐藏测试点
- AI 生成结果需直接用于创建题目和判题数据

## 4. 非功能需求

- 前后端分离
- 支持后续扩展更多语言和判题策略
- 高内聚、低耦合，判题服务和 AI 服务可替换
- 数据持久化采用 PostgreSQL
- 界面布局接近 LeetCode 的三栏工作区

## 5. 后续建议

- 增加用户系统、登录鉴权与权限控制
- 增加隐藏测试点和批量导入
- 判题执行迁移到容器沙箱
- 增加异步队列与提交记录页
- 增加题目标签、难度筛选与搜索
